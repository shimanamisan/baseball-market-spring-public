## 分析・判断記録

### なぜ `thymeleaf-extras-springsecurity6` を追加したか

Spring Security 6 系と Thymeleaf を併用すると、テンプレート側で `sec:authorize="isAuthenticated()"` などの認可判定や `${#authentication.principal}` 参照が必要になる。フェーズ 4 以降のヘッダ／マイページ画面で確実に使うため、依存だけ先に入れた。

### なぜ `app.uploads.path` を `static/uploads` にしたか／リスク

旧 PHP は `public/uploads/` 配下に直接画像を置いていた。互換性のため dev では `app/src/main/resources/static/uploads/` を指定し、Thymeleaf から `/uploads/xxx.jpg` で参照可能にする。

ただし **jar 化すると static は read-only** になるため、本番ビルドでは外部ディレクトリ + `WebMvcConfigurer#addResourceHandlers("/uploads/**")` 経由の公開に切り替える必要がある。フェーズ 5-3（プロフィール画像）／フェーズ 6-4（商品画像）の実装時に最終判断する。

### Flyway baseline-version の選定

`baseline-version=0` を指定したのは、後続マイグレーション `V1__init.sql` を「未適用扱い」にできるようにするため。旧 DB に対しては V1 が「ALREADY EXISTS（実体は baseline と同等）」となるか、新規環境では V1 が新規適用されるか、どちらの経路でも整合する想定。フェーズ 2 で実環境動作を検証する。
